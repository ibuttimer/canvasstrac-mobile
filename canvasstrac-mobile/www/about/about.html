Overview

This application was developed for the Capstone Project of the Full Stack Web Development Specialization set of courses provided by coursera.


Functionality

The purpose of the CanvassTrac application suite is to provide an application that may be utilised during elections to allow a campaign to collect and track the results of canvasses.



The application suite consists of three individual elements:

  • A server application
  • A management web application
    To facilitate the creation and configuration of canvasses and their associated surveys.
  • A mobile client application
    For the use of canvassers on the ground to collect voter data.

The list of currently supported features of the web application are as follows:

  • User login
  • User management
    Including user creation and control.
  • Canvass configuration
    Canvasses may be created and configured, allowing the assignment of addresses to individual canvassers.
  • Survey configuration
    Allowing the configuration of a set of specific questions to gather answers on.
  • Canvass result analysis
    Providing details of the results collected by the canvassers.
  • Feedback submission
  • Notice configuration
    Allowing the configuration of notices for display on the client applications.
  • Automatic logout on idle timeout.

The list of currently supported features of the mobile application are as follows:

  • User login
  • Download of assigned canvass route and survey information.
  • Mapping service
    To allow canvassers to quickly and efficiently cover their assigned canvass route.
  • Survey response entry
    In order to allow the campaign team to accurately monitor campaign progress.
